# Samsung Local API Notes

The bridge talks to the Samsung Soundbar local IP-control endpoint:

```text
https://<SOUNDBAR_HOST>:1516/
```

The endpoint commonly uses a local/self-signed TLS certificate, so `SOUNDBAR_VERIFY_SSL=false` is the default for LAN-only deployments.

## Token types

There are two different tokens:

- `BRIDGE_AUTH_TOKEN`: user-generated secret that protects the bridge HTTP endpoints.
- Samsung `AccessToken`: generated by the bridge through `createAccessToken`, cached internally, and sent only to the soundbar local API.

Users normally do not need to copy or store the Samsung `AccessToken`.

## Common read methods

The bridge currently uses the local JSON-RPC API for:

- `createAccessToken`
- `getVolume`
- `getMute`
- `soundModeControl` readback

## Common control methods

The bridge maps HTTP endpoints to Samsung local API calls such as:

- `remoteKeyControl` for volume, mute, source and power actions
- `soundModeControl` for sound-mode changes

Power and sound-mode behavior may vary by model and firmware. Treat those endpoints as model-verified, not universal Samsung guarantees.
