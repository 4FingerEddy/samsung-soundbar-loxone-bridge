# Generic Portainer Deployment Quickstart

Status: generic deployment helper. No secrets in this file.

## What the user must provide

The user needs exactly two model/site-specific values:

```text
SOUNDBAR_HOST=<soundbar IP address or DNS name>
BRIDGE_AUTH_TOKEN=<random private bridge token>
```

Everything else has safe defaults for the Samsung local IP control path.

## 1. Find and set the Soundbar IP

Recommended: create a DHCP reservation/static lease in the router for the Samsung Soundbar, then use that reserved IP.

Common places to find it:

- Router / DHCP client list
- UniFi client list
- Samsung SmartThings device details
- Network scanner on the local LAN

Set it in Portainer stack environment variables:

```env
SOUNDBAR_HOST=192.168.1.123
```

The stack template consumes it via:

```yaml
SOUNDBAR_HOST: "${SOUNDBAR_HOST}"
```

## 2. Generate BRIDGE_AUTH_TOKEN without command line skills

The bridge token is not provided by Samsung. It is just a private random password for the bridge HTTP API.

Simple option for non-technical users:

1. Open this file locally in a browser:

   ```text
   deployment/token-generator.html
   ```

2. Click **Token erzeugen**.
3. Click **Kopieren**.
4. Paste it into Portainer as:

   ```env
   BRIDGE_AUTH_TOKEN=<copied token>
   ```

The token generator uses the browser's Web Crypto API and sends nothing to the internet.

Alternative: use a password manager's built-in password generator and create a random value with at least 32 characters. Use letters/numbers and avoid spaces.

## 3. Portainer environment variables

Minimum useful set:

```env
SOUNDBAR_HOST=192.168.1.123
SOUNDBAR_PORT=1516
SOUNDBAR_SCHEME=https
SOUNDBAR_VERIFY_SSL=false
BRIDGE_PORT=8088
BRIDGE_AUTH_TOKEN=<generated token>
DEBUG_ENDPOINTS=false
```

Full example:

```text
deployment/portainer.env.example
```

## 4. Samsung AccessToken clarification

There are two tokens:

- `BRIDGE_AUTH_TOKEN`: generated by the user; protects the bridge endpoints and is used by Loxone.
- Samsung `AccessToken`: generated/cached automatically by the bridge via Samsung `createAccessToken`; the user does not enter this anywhere.

Do not paste the Samsung AccessToken into Loxone or Portainer.

## 5. Loxone placeholders

In the Loxone XML templates replace:

```text
REPLACE_WITH_BRIDGE_IP_PORT
REPLACE_WITH_BRIDGE_AUTH_TOKEN
```

Example:

```text
REPLACE_WITH_BRIDGE_IP_PORT -> 192.168.1.10:8088
REPLACE_WITH_BRIDGE_AUTH_TOKEN -> the generated bridge token
```

## 6. First checks after deploy

Read-only process check:

```text
http://<bridge-ip>:8088/health
```

Read-only Loxone status endpoint:

```text
http://<bridge-ip>:8088/api/v1/loxone/status.txt?token=<BRIDGE_AUTH_TOKEN>
```

Control endpoints such as volume/source/mode have real physical effect on the Soundbar. Test those deliberately, not accidentally.
